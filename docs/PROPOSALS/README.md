# Proposals

Speculative ideas raised by the **creative-director** live here. Each file is
a single self-contained proposal in the format described in
`agents/creative-director.md`.

## Lifecycle

- `draft` — being written
- `under-review` — awaiting verdicts from the listed reviewers (CEO + at
  least one of CTO / Head of Product / Head of Growth / Director of Data)
- `approved` — every reviewer endorsed; CEO lifts it into the plan or the
  founder queue
- `rejected` — one or more reviewers said no; the file stays here as a
  record, and the same idea is not re-filed as a new document
- `shipped` — approved and delivered (still linked here for provenance)

## Not for

- Backlog items that already fit the current roadmap — those go straight
  into `docs/BACKLOG.md` via the relevant role
- Founder approvals — those go into `docs/PENDING_APPROVALS.md`; only the
  CEO lifts a proposal there after review passes
- Anything the creative director can execute directly — nothing, by design.
  Every proposal ships through someone else's hands.
