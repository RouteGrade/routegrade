Speculative ideas raised by the **creative-director** used to live here as
individual markdown files. Each proposal is now a row in the Proposals
database in Notion instead, in the format described in
`agents/creative-director.md`.

## Lifecycle

- `draft` — being written
- `under-review` — awaiting verdicts from the listed reviewers (CEO + at
  least one of CTO / Head of Product / Head of Growth / Director of Data)
- `approved` — every reviewer endorsed; CEO lifts it into the plan or the
  founder queue
- `rejected` — one or more reviewers said no; the row stays as a record, and
  the same idea is not re-filed as a new row
- `shipped` — approved and delivered (still linked there for provenance)

## Not for

- Backlog items that already fit the current roadmap — those go straight
  into the Backlog database in Notion via the relevant role
- Founder approvals — those go into the Pending Approvals database in
  Notion; only the CEO lifts a proposal there after review passes
- Anything the creative director can execute directly — nothing, by design.
  Every proposal ships through someone else's hands.
